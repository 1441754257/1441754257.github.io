大语言模型基础-从Transformer到LLM核心原理

本文系统梳理大语言模型的核心原理，从 N-gram 统计模型到 Transformer 架构，再到提示工程与模型局限性，帮助开发者建立完整的 LLM 知识体系。

一、语言模型演进历史

1.1 统计语言模型：N-gram

语言模型（Language Model） 的根本任务是计算一个词序列出现的概率。

N-gram 模型基于马尔可夫假设：一个词的出现概率只与它前面有限的 n-1 个词有关。

Bigram（N=2）示例计算：

语料库：datawhale agent learns / datawhale agent works

N-gram 的致命缺陷：

  • 数据稀疏性：未见过的词序列概率为 0
  • 泛化能力差：无法理解词语间的语义相似性（agent 和 robot 被视为完全不同的词）

1.2 神经网络语言模型与词嵌入

2003 年 Bengio 提出前馈神经网络语言模型，核心思想：

  1. 词嵌入（Word Embedding）：将词映射为高维连续向量，语义相近的词在向量空间中距离更近

1.3 RNN 与 LSTM

循环神经网络（RNN） 引入隐藏状态作为记忆，解决了固定窗口限制。

但 RNN 存在长期依赖问题：

  • 梯度消失：长序列中早期信息无法有效传递
  • 梯度爆炸：梯度值变得极大

LSTM（长短时记忆网络） 通过门控机制解决此问题：

  • 遗忘门：决定丢弃哪些历史信息
  • 输入门：决定存入哪些新信息
  • 输出门：决定输出哪些信息

二、Transformer 架构详解

2017 年 Google 提出 Transformer，完全抛弃循环结构，依赖注意力机制实现并行计算。

2.1 Encoder-Decoder 整体结构

  • 编码器：理解输入，为每个词元生成富含上下文信息的向量
  • 解码器：参考编码器输出，逐词生成目标序列

2.2 自注意力机制（Self-Attention）

核心思想：处理每个词时，动态关注句子中所有其他词，分配不同的注意力权重。

三个角色：

  • Query（Q）：当前词，主动查询其他词
  • Key（K）：被查询词的”标签”
  • Value（V）：词本身携带的”内容”

计算公式：

计算步骤：

  1. 生成 Q、K、V 向量（原始词向量 × 权重矩阵）
  2. 计算相关性得分：Q 与所有 K 做点积
  3. 除以 √dₖ 缩放，防止梯度过小
  4. Softmax 归一化得到注意力权重
  5. 加权求和 V 向量，得到新的词表示

2.3 多头注意力（Multi-Head Attention）

思想：让多个”专家”从不同角度审视句子，捕捉不同类型的关系（指代、时态、从属等）。

2.4 前馈神经网络（FFN）

每个 Encoder/Decoder 层中，注意力层之后跟着逐位置前馈网络：

  • “先扩大再缩小”：d_ff = 4 × d_model，学习更丰富的特征
  • 逐位置：对每个词元独立处理，所有位置共享权重

2.5 残差连接与层归一化

每个子模块都被 Add & Norm 包裹：

  • 残差连接（Add）：解决梯度消失，梯度可绕过子模块直接传播
  • 层归一化（Norm）：稳定训练，加速收敛

2.6 位置编码（Positional Encoding）

自注意力本身不包含位置信息，需要额外注入：

将位置编码加到词嵌入向量上，使相同词在不同位置有不同表示。

2.7 Decoder-Only 架构（GPT 系列）

核心思想：语言的本质就是预测下一个词，无需编码器。

自回归生成过程：

  1. 输入起始文本：”Datawhale Agent is”
  2. 预测下一词：”a”
  3. 将 “a” 加入输入，继续预测：”powerful”
  4. 循环直到生成完整句子

掩码自注意力（Masked Self-Attention）：

  • 在注意力分数矩阵中，将当前位置之后的词对应分数设为 -∞
  • Softmax 后这些位置概率为 0
  • 确保预测时只能看到已生成的内容

Decoder-Only 的优势：

  • 训练目标统一：只需”预测下一个词”
  • 结构简单，易于扩展到千亿参数
  • 天然适合生成任务（对话、写作、代码）

三、提示工程（Prompt Engineering）

3.1 模型采样参数

参数 作用 推荐值
Temperature 控制输出随机性 事实任务 < 0.3；创意任务 0.7~1.5
Top-k 只从概率最高的 k 个词中采样 k=1 退化为贪心采样
Top-p 累积概率达到 p 的最小词集合中采样 比 Top-k 更能适应分布变化

参数优先级：Temperature → Top-k → Top-p

注意：Temperature=0 时，Top-k 和 Top-p 失效；Top-k=1 时，其他参数失效。

3.2 三种提示策略

零样本提示（Zero-shot）：直接下达指令，不提供示例

单样本提示（One-shot）：提供一个示例

少样本提示（Few-shot）：提供多个示例，覆盖不同情况，效果最佳

四、文本分词（Tokenization）

4.1 为什么需要子词分词

方式 问题
按词分词 词表爆炸、未登录词（OOV）问题
按字符分词 单字符无语义，学习效率低
子词分词 ✅ 兼顾词表大小和语义表达

子词分词核心思想：常见词保留完整，罕见词拆分为有意义的子词片段。

4.2 BPE 算法（Byte-Pair Encoding）

GPT 系列采用的分词算法，核心是贪心合并：

  1. 初始化：词表 = 所有基本字符
  2. 统计相邻词元对的出现频率
  3. 合并频率最高的词元对，加入词表
  4. 重复直到词表达到预设大小

相关算法：

  • WordPiece（BERT）：合并标准是最大化语言模型概率
  • SentencePiece（Llama）：将空格视为普通字符，支持任意语言

4.3 分词对开发者的影响

  • 上下文窗口：以 Token 数量计算，中文比英文消耗更多 Token
  • API 成本：按 Token 计费，了解分词有助于控制成本
  • 模型异常：2+2 和 2 + 2 可能被分成不同 Token，影响计算结果

五、缩放法则与模型局限性

5.1 缩放法则（Scaling Laws）

模型性能与参数量、数据量、计算量呈幂律关系：持续增加这三个要素，性能可预测地平滑提升。

Chinchilla 定律（2022）：

  • 最优模型参数量和训练数据量存在最优配比
  • 数据量应与参数量等比例增加
  • 纠正了”越大越好”的片面认知

能力涌现（Emergent Abilities）：

  • 模型规模达到阈值后，突然出现小模型中不存在的能力
  • 如：链式思考、指令遵循、多步推理、代码生成
  • 这些能力在参数量达到数百亿后才显著出现

5.2 模型幻觉（Hallucination）

定义：模型生成与客观事实矛盾，或编造不存在的信息。

幻觉类型：

  • 事实性幻觉：生成与现实不符的信息
  • 忠实性幻觉：摘要/翻译未忠实反映原文
  • 内在幻觉：生成内容与输入信息矛盾

产生原因：

  • 训练数据包含错误信息
  • 自回归生成机制无内置事实核查
  • 复杂推理中逻辑链条出错

缓解方法：

方法 原理
RAG（检索增强生成） 生成前从外部知识库检索相关信息作为上下文
多步推理与验证 引导模型逐步推理，每步自我检查
外部工具调用 调用搜索引擎、计算器获取实时准确信息
RLHF 通过人类反馈强化学习，减少有害输出
